# Canonical catalog schema (Phase 1)

Processed output: `data/processed/restaurants.parquet` (generated by `scripts/ingest_zomato.py`).

Source dataset: [ManikaSaini/zomato-restaurant-recommendation](https://huggingface.co/datasets/ManikaSaini/zomato-restaurant-recommendation).

## Columns

| Column | Type | Description |
|--------|------|-------------|
| `id` | string | Stable 16-char hex from SHA-256 of `url\|name\|address` |
| `name` | string | Restaurant name |
| `city` | string | Metro inferred from Zomato URL path or address (normalized via `config.yaml` aliases) |
| `locality` | string | Main listing area: `listed_in(city)` when set, else source `location` |
| `cuisines` | list\<string\> | Split from source `cuisines` (comma-separated) |
| `rating` | float or null | Parsed from `rate` (e.g. `4.1/5`); null if missing/unparsable |
| `cost_for_two` | int or null | INR digits from `approx_cost(for two people)` |
| `budget_tier` | string or null | `low` / `medium` / `high` from INR thresholds in `config.yaml` |
| `votes` | int or null | Review vote count |
| `address` | string or null | Full address |
| `url` | string or null | Zomato URL |
| `raw_features` | string or null | JSON blob: `online_order`, `book_table`, `rest_type`, `listed_in_*`, `dish_liked` |

```json
  "raw_features": "{\"online_order\": \"Yes\", \"book_table\": \"Yes\", \"rest_type\": \"Casual Dining\", \"listed_in_type\": \"Buffet\", \"listed_in_city\": \"Banashankari\", \"dish_liked\": \"Pasta, Lunch Buffet, ...\"}"
}
```

*(Example `id` shortened for illustration; real ids are 16 hex characters.)*

## Reproducing the catalog

From the repository root (after `pip install -e .` or with `PYTHONPATH=src`):

```bash
python scripts/ingest_zomato.py --config config.yaml
```

Artifacts:

- `data/raw/zomato_hf_snapshot.parquet` — raw HF columns
- `data/processed/restaurants.parquet` — canonical schema
- `data/processed/ingest_report.json` — row counts and drop reasons

Tune `rating`, `budget_tiers_inr`, and `city_aliases` in `config.yaml`.

**API note:** User-facing recommendations use **`budget_max_inr`** (max cost for two) for filtering, not the `low` / `medium` / `high` tier labels. Tiers remain on each row for display context.
